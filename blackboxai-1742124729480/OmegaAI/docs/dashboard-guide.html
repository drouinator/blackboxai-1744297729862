OmegaAI Dashboard Guide

Complete guide for managing your AI Server infrastructure

Quick Start

1. Accessing the Dashboard

The dashboard can be accessed in two ways:

  • Directly via http://localhost:8000/OmegaAI/dashboard.html
  • Through the OmegaAI Interface: Click on the Dashboard icon in the sidebar

2. Initial Setup

Ensure your AI Server and Pi 5 are properly configured:

  • Verify SSH keys are in place
  • Confirm static IP configuration
  • Check service scripts are executable

Features Guide

Resource Monitoring

The dashboard provides real-time monitoring of system resources:

  • CPU Usage: View current and historical CPU utilization
  • RAM Usage: Monitor memory consumption
  • GPU Usage: Track GPU resource allocation
  • Storage: Monitor connected drives and space usage

SSH Configuration

Manage SSH connections between your Pi 5 and AI Server:

  • View connection status
  • Manage SSH keys
  • Configure security settings
  • Test connections

Service Management

Control AI services and system processes:

  • Start/Stop AI Services
  • Monitor service health
  • View service logs
  • Configure automatic restarts

Best Practices

Resource Management

  • Monitor resource usage regularly
  • Set up alerts for critical thresholds
  • Schedule maintenance during low-usage periods

Security

  • Regularly update SSH keys
  • Monitor access logs
  • Keep system packages updated

Troubleshooting

Common Issues

Service Not Starting

Check:

  • Service logs for errors
  • Resource availability
  • Permission settings

SSH Connection Issues

Verify:

  • Network connectivity
  • SSH key permissions
  • Firewall settings
